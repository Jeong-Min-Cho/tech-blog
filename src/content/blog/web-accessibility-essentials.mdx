Web accessibility isn't just about compliance—it's about ensuring everyone can use your website, regardless of their abilities. Let's explore practical techniques to make your sites more inclusive.

## Why Accessibility Matters

Over 1 billion people worldwide have some form of disability. This includes:

- **Visual impairments** - Blindness, low vision, color blindness
- **Hearing impairments** - Deafness, hard of hearing
- **Motor impairments** - Limited fine motor control, paralysis
- **Cognitive impairments** - Dyslexia, ADHD, memory issues

Beyond the ethical imperative, accessible websites also:
- Rank better in search engines
- Work better on mobile devices
- Are more usable for everyone

## The POUR Principles

WCAG (Web Content Accessibility Guidelines) is built on four principles:

### 1. Perceivable

Users must be able to perceive the content.

```html
<!-- Bad: Image without description -->
<img src="chart.png">

<!-- Good: Descriptive alt text -->
<img src="chart.png" alt="Sales increased 25% from Q1 to Q2 2024">

<!-- Good: Decorative images -->
<img src="decorative-line.png" alt="" role="presentation">
```

### 2. Operable

Users must be able to operate the interface.

```html
<!-- Bad: Click-only interaction -->
<div onclick="openMenu()">Menu</div>

<!-- Good: Keyboard accessible button -->
<button onclick="openMenu()">Menu</button>
```

### 3. Understandable

Users must be able to understand the content.

```html
<!-- Good: Clear form labels -->
<label for="email">Email Address</label>
<input type="email" id="email" name="email">

<!-- Good: Error messages that explain the issue -->
<p role="alert" class="error">
  Please enter a valid email address (e.g., name@example.com)
</p>
```

### 4. Robust

Content must work with current and future technologies.

```html
<!-- Good: Semantic HTML -->
<nav aria-label="Main navigation">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
  </ul>
</nav>
```

## Common Accessibility Issues (and Fixes)

### Missing Skip Links

Skip links help keyboard users bypass repetitive navigation.

```html
<body>
  <a href="#main-content" class="skip-link">
    Skip to main content
  </a>
  <nav><!-- navigation here --></nav>
  <main id="main-content">
    <!-- main content -->
  </main>
</body>
```

```css
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  padding: 8px;
  background: #000;
  color: #fff;
  z-index: 100;
}

.skip-link:focus {
  top: 0;
}
```

### Poor Color Contrast

Text must have sufficient contrast against its background.

```css
/* Bad: 2.5:1 ratio - fails WCAG */
.low-contrast {
  color: #767676;
  background: #ffffff;
}

/* Good: 4.5:1 ratio - passes WCAG AA */
.good-contrast {
  color: #595959;
  background: #ffffff;
}

/* Better: 7:1 ratio - passes WCAG AAA */
.high-contrast {
  color: #333333;
  background: #ffffff;
}
```

### Missing Focus Indicators

Never remove focus outlines without providing an alternative.

```css
/* Bad: Removes all focus indication */
*:focus {
  outline: none;
}

/* Good: Custom focus styles */
:focus-visible {
  outline: 2px solid #2563eb;
  outline-offset: 2px;
}
```

### Inaccessible Custom Components

When building custom components, include proper ARIA attributes.

```html
<!-- Accessible accordion -->
<div class="accordion">
  <h3>
    <button
      aria-expanded="false"
      aria-controls="section1"
      id="accordion1"
    >
      Section 1
    </button>
  </h3>
  <div
    id="section1"
    role="region"
    aria-labelledby="accordion1"
    hidden
  >
    <p>Content goes here...</p>
  </div>
</div>
```

```javascript
button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', !expanded);
  panel.hidden = expanded;
});
```

## Testing for Accessibility

### Automated Testing

Use these tools to catch common issues:

- **axe DevTools** - Browser extension for automated testing
- **Lighthouse** - Built into Chrome DevTools
- **WAVE** - Web accessibility evaluation tool

### Manual Testing

Automated tools only catch about 30% of issues. Also test:

1. **Keyboard navigation** - Tab through your entire site
2. **Screen reader** - Use NVDA (Windows) or VoiceOver (Mac)
3. **Zoom** - Test at 200% and 400% zoom
4. **Color blindness** - Use simulation tools

### Screen Reader Testing Checklist

| Element | What to Check |
|---------|--------------|
| Images | Alt text is announced |
| Links | Purpose is clear out of context |
| Forms | Labels are associated with inputs |
| Headings | Hierarchy makes sense |
| Tables | Headers are properly marked up |

## Quick Wins

Here are improvements you can make today:

1. **Add `lang` attribute** to your HTML
   ```html
   <html lang="en">
   ```

2. **Use semantic HTML** - `<button>`, `<nav>`, `<main>`, `<article>`

3. **Add alt text** to all meaningful images

4. **Ensure 4.5:1 color contrast** for normal text

5. **Make focus states visible**

6. **Add `aria-label`** to icon-only buttons
   ```html
   <button aria-label="Close menu">
     <svg><!-- X icon --></svg>
   </button>
   ```
